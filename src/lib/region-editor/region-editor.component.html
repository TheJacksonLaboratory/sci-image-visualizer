The Regions tab lists every region drawn on the currently displayed image. Regions are tracked per image: switching to another image hides these and shows that image's regions; switching back restores them. Edits in the table apply live to the diagram — there is no save or cancel.

Creating regions

Use the diagram toolbar tools to draw new regions: rectangle, polyline, freeform path, brush, or wand. Anything you draw appears immediately in the table, with the class name "legend" by default.

The table columns

  • Class — the region's class label. Click the to rename it inline.
  • Color — the region's outline colour. Pick a colour to recolour that region live on the diagram.
  • Surface — the region's area, in µm²/mm² when the image carries a physical pixel size, otherwise in px². Holes (donuts) and multi-part regions are accounted for.
  • The trash icon at the end of a row deletes that one region.
  • Use "x512" to round all rectangle widths/heights to the nearest multiple of 512.

Region geometry is edited on the diagram itself (drag vertices, the vertex tools, brush/wand, and the right-click region actions), not in this table.

Selecting rows

  • Click a row to select it. The corresponding region is highlighted in the diagram (Plotly shows its edit handles; the OpenSeadragon image view highlights it and shows its handles in select mode).
  • Ctrl/Cmd+click to add or remove a row from the selection. Shift+click to extend the selection in a range.
  • Selecting a region in the diagram highlights the matching row in the table.
  • Clicks on the colour picker, the trash icon, or the active class-edit input do not change the selection — that's standard PrimeNG behaviour.

Bulk actions on the selected rows

  • Select every region in the table at once (also a right-click action on the image).
  • Edit the label on every selected row at once.
  • Delete every selected region — both from the table and the diagram.
  • Delete every region on this image.

Colours

  • One region — the Color column sets a single region's outline colour (see "The table columns" above).
  • Selected regions — select one or more rows, then use the palette button to set the outline colour of all the selected regions at once.

Region actions (right-click the diagram)

Right-click a region — or the current multi-selection — on the diagram for the "Selected region(s)" actions:

  • Merge / group — combine 2+ selected regions into one (it may end up multi-part if the pieces don't touch).
  • Ungroup — split a multi-part region back into separate regions.
  • Inverse — replace the selection with everything outside it (the regions become holes in the image rectangle).
  • Simplify — reduce the vertex count (Douglas–Peucker) by a pixel threshold; pick a preset or a custom value.
  • Convert to Bézier / to polygon and Delete.

Display options

  • Toggle showing class labels on every region in the diagram (slower with many regions).

Import / export

  • Import regions from a GeoJSON file.
  • Export the current image's regions to a GeoJSON file. (The per-image cache is in-memory only — exporting is how you persist regions across sessions.)
